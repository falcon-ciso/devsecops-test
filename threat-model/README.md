# Threat Model: Микросервис на Python/Go + CI/CD + Kubernetes

## 1. Компонент: API Gateway (Ingress / Service)

| Угроза (STRIDE) | Описание | Последствия | Контрмеры | Приоритет |
|---|---|---|---|---|
| **S (Spoofing)** | Кража JWT-токена из логов или метрик | Вход под чужим аккаунтом | Короткое время жизни JWT, HTTPS, маскировка в логах | **Critical** |
| **T (Tampering)** | Изменение параметров запроса (например, `user_id` в URL) | Доступ к чужим данным | Проверка авторизации на сервере (не доверять клиенту) | **High** |
| **R (Repudiation)** | Атака не оставляет следов | Нельзя определить источник атаки | Аудит логов (доступа, ошибок), их долгосрочное хранение | **Medium** |
| **I (Info Disclosure)** | Утечка стека ошибок или версий библиотек | Быстрый подбор эксплойтов | Настройка error handling, скрытие версий в ответах | **High** |
| **D (DoS)** | Отправка огромного количества запросов | Сервис недоступен | Rate limiting, лимиты на размер запроса | **Medium** |
| **E (Elevation)** | Подмена `user_id=123` на `user_id=0` (админ) | Получение прав администратора | Проверка прав на каждый запрос, минимум привилегий | **Critical** |

## 2. Компонент: CI/CD Pipeline (GitLab CI / GitHub Actions)

| Компонент | Угроза | Контрмера | Приоритет |
|---|---|---|---|
| **CI-скрипты (.gitlab-ci.yml)** | Вредоносные изменения в пайплайне | Code review для CI-файлов, подпись коммитов | **Critical** |
| **Переменные окружения** | Утечка `KUBECONFIG`, `AWS_SECRET_KEY` из логов | Masked переменные, хранение в Vault | **Critical** |
| **Образы (Trivy, Semgrep)** | Использование вредоносного образа (как с Trivy v0.69.4) | Фиксированные версии (не latest), разрешенный registry | **High** |
| **Артефакты (образы)** | Компрометация собранного образа после сканирования | Подпись контейнеров (Cosign), сканирование перед деплоем | **High** |

## 4. Компонент: Видеопоток с камер (RTSP / WebRTC / S3-совместимое хранилище)

| Угроза (STRIDE) | Описание | Последствия | Контрмеры | Приоритет |
|---|---|---|---|---|
| **I (Info Disclosure)** | Перехват видеопотока (RTSP без TLS, открытый S3 бакет) | Утечка видео: номера, лица, геолокация, сцены | **Шифрование потока (SRTP/TLS)**, закрытые бакеты, подписанные URL | **CRITICAL** |
| **T (Tampering)** | Подмена видеокадра (подставить другой номер, вставить ложное событие) | Ложное распознавание, штраф невиновному, пропуск нарушителя | **Цифровая подпись кадра**, проверка целостности в ML-пайплайне | **HIGH** |
| **D (DoS)** | Отключение камеры, забивание видеоканала мусором | Потеря событий, невозможность распознавания | Оповещение о потере потока, резервные каналы (4G), rate limiting | **MEDIUM** |

## 5. Компонент: ML-модель (распознавание номеров, возможно лиц)

| Угроза | Описание | Последствия | Контрмеры | Приоритет |
|---|---|---|---|---|
| **Атака на модель (Adversarial ML)** | Специальный стикер на номере, который заставляет модель ошибаться | Номер распознан неверно (например, «А777ВС» → «А777СС») | **Робастность модели**, проверка в реальном времени (ensemble), логирование сомнительных срабатываний | **HIGH**|
| **Отравление данных (Data Poisoning)** | Подмена фото номеров в тренировочной выборке | Модель обучается на неверных данных, начинает ошибаться системно | **Контроль версий датасетов**, контроль доступа к S3 с датасетами, проверка на аномалии | **CRITICAL**|
| **Кража модели** | Выгрузка весов модели (файл .h5 / .pth) из S3 или Git | Конкуренты получают ноу-хау, могут найти уязвимости офлайн | **Шифрование модели**, контроль доступа к S3, аудит, шифрование дисков | **HIGH**|

## 6. Компонент: База номеров (история распознаваний)

| Угроза | Описание | Последствия | Контрмеры | Приоритет |
|---|---|---|---|---|
| **Утечка истории** | Неавторизованный доступ к БД (SQL-инъекция, кража пароля) | Штрафы по 152-ФЗ (обработка ПДн, номера авто — персональные данные?), репутационные риски | **SAST/SCA** (ищем SQLi), шифрование номеров в БД, аудит доступа, эскейп | **CRITICAL**|
| **Уничтожение истории** | Атака ransomware на БД | Потеря данных о проходах за период, невозможность восстановления событий | Регулярные бекапы (offsite, на отдельное S3), WORM-бакеты | **MEDIUM**|
